Quick Start

Quick Start Guide

Get started with YMD/PMD in 5 minutes.

Installation

Step 1: Clone Repository

git clone https://github.com/daviguides/ymd-spec.git
cd ymd-spec

Step 2: Run Installer

./install.sh

This installs specifications to ~/.claude/ymd-spec/.

Step 3: Verify Installation

ls ~/.claude/ymd-spec/

You should see:

  • ymd_format_spec.md
  • pmd_format_spec.md
  • composition_spec.md
  • context/
  • prompts/
  • cheatsheet/

Your First YMD File

Create Minimal YMD

Create my_first_prompt.ymd:

meta:
  id: my_first_prompt
  kind: task
  version: 1.0.0
  title: My First Prompt

user: |
  {{ task }}

Key points:

  • ✅ Metadata grouped under meta:
  • ✅ At least one section (here: user)
  • ✅ Variables use {{ variable }}

Test It

If using ymd-prompt Python library:

from ymd_prompt import render_ymd

result = render_ymd(
    "my_first_prompt.ymd",
    variables={"task": "Explain Python generators"}
)
print(result)

Your First PMD Component

Create Simple Role

Create components/expert.pmd:

You are an expert in {{ domain }}.

Your approach:
- Provide clear, technical answers
- Focus on best practices
- Use concrete examples

Key points:

  • ❌ NO meta: section (PMDs don’t have metadata!)
  • ✅ Pure Markdown + Jinja2
  • ✅ Document expected variables in comment

Use It in YMD

Update your YMD:

meta:
  id: my_first_prompt
  kind: task
  version: 1.0.0
  title: My First Prompt

system: |
  {% include "components/expert.pmd" %}

user: |
  {{ task }}

Now the prompt includes the expert role!

Common Patterns

Pattern 1: Simple Task

meta:
  id: simple_task
  kind: task
  version: 1.0.0
  title: Simple Task

user: |
  {{ task }}

Pattern 2: With Role

meta:
  id: with_role
  kind: task
  version: 1.0.0
  title: Task with Role

system: |
  {% include "roles/expert.pmd" %}

user: |
  {{ task }}

Pattern 3: Custom Sections

meta:
  id: custom_sections
  kind: task
  version: 1.0.0
  title: Custom Sections

context: |
  Working on {{ project_name }}

constraints: |
  - Keep code simple
  - Add error handling

user: |
  {{ task }}

Pattern 4: Conditional

meta:
  id: conditional
  kind: task
  version: 1.0.0
  title: Conditional

system: |
  {% if level == "beginner" %}
  {% include "roles/mentor.pmd" %}
  {% else %}
  {% include "roles/expert.pmd" %}
  {% endif %}

user: |
  {{ task }}

Validation

Validate Your Files

/validate-composition path/to/file.ymd

Or with the agent:

@ymd-validator Check my code_review.ymd

Common Issues

Issue: PMD has metadata

<!-- ❌ WRONG -->
meta:
  id: component

Fix: Remove meta: - only YMD has metadata!

Issue: Flat metadata in YMD

# ❌ WRONG
id: example
kind: task

Fix: Group under meta:

# ✅ CORRECT
meta:
  id: example
  kind: task

Issue: Include not found

{% include "expert.pmd" %}  # Can't find it!

Fix: Use correct relative path

{% include "components/expert.pmd" %}
